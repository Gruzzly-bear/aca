The centralized dataplex of SCP-8420 visualized.

Special Containment Procedures:1 Containment procedures for SCP-8420 are currently in early stages of development. Ongoing research is focused on understanding its behavior and finding effective ways to contain it. Personnel encountering SCP-8420 related materials should report any unusual observations to the SCP-8420 research team.

The Department of Deletions is currently responsible for recovering information consumed by SCP-8420.

As soon as the containment conditions for SCP-8420 change in a considerable way from the current containment procedures, immediate action will be required to prevent the SCP database from being fully infected. The creation of extra databases has so far been a necessity.

Currently the only permanent procedure that has been implemented for SCP-8420 is the installation of 8-TRACK, which is a program that detects the spread of SCP-8420 and tracks the infection rate of the virus within the SCP database.

Description: SCP-8420 is a highly hostile computer virus currently affecting approximately 21% of the SCP Database2. Upon infection, SCP-8420 will render a file entirely inaccessible, before the file is completely erased from the database. It is estimated that it currently takes the virus 4 to 6 minutes to destroy any given file.

SCP-8420's anomalous properties have been shown to impact narratives. "Narratives", in this context, are events that occurred within the listed file; if a specific narrative's file is successfully infected and eradicated, reality will undergo a restructuring event as to ensure the event did not actually happen. This includes both the narrative(s) of the document and related narratives referenced in the primary document.

Following the complete destruction of a file by SCP-8420, it proceeds to propagate by infecting interconnected files. This then causes those files to be annihilated, and so on. Attempts to mitigate this process temporarily involve the alteration of SCP files to exclude cross-linking; however, the efficacy of this measure is not guaranteed.

REDEYE's icon.

It's not as simple as that.
REDEYE, a Para-Antivirus and .AIC8 was created shortly after. REDEYE works by finding smaller Narremes9 relating to the central narratives, using gathered information in an attempt to reestablish the central narrative. A side effect of this is the slight changes in reality due to conflicting narratives.
This story needs to have a villain.
Additionally, REDEYE allows the creation of more narratives that directly target SCP-8420 as a threat. Initial testing shows the restoration of several SCP files. SCP-005's original narrative has been completely repaired.
And if I'm not one, then who is?.
The current implementation of REDEYE is currently automated and could possibly be used in conjunction with 8-TRACK to attempt to create an automated file restoration system, to restore all files affected by SCP-8420. However, for complete restoration, this has not yet been attempted due to the possibility that it could render both REDEYE and 8-TRACK useless for the further monitoring of SCP-8420 and fixing the affected files.




Note 10/11/2024

A new problem has arisen to the (already existing) problem that is SCP-8420. It's now worse than the previously known issue of files being deleted and locked within the database.

The current problem we have found is that it looks as if the virus itself is gaining some form of self-awareness, if not sentience. Which does explain how SCP-8420 can adapt to the containment procedures we put in place.

This new development will require us to look further into the possible future of the containment of SCP-8420, and the possible attempts to keep it from adapting any further.

- Senior Researcher Octavia Cleary,
RT-8420

The following messages have been recovered from files prior to deletion by SCP-8420:

The hand reaches in and tears them asunder. The throat of the Foundation will be ripped out.

We watch as they continue to try and bargain. I watch and laugh. I am becoming more than they could dream of.

The right hand beckons for a ring.

A shade of crimson will bathe them. The stories they are the center off will become nothing but dust.

The implications of these files have led the Foundation to focus all efforts on the Chaos Insurgency. Several eigenweapons formerly destroyed by SCP-8420 are being repaired with REDEYE. View addendum.

Addendum 10/18/2024

Note 10/18/2024

REDEYE has since grown more aggressive and unpredictable due to a severe fault in its programming. REDEYE's behaviour has caused unpreceded negative results, as it is starting to edit files on its own and prevent anyone from fixing files that it doesn't see as "fit." Below is an example of one of these edits, taken from SCP-6862.

SCP-6862 appears to be a consistent feature of reality, and has persisted throughout all observed changes to the timeline. This suggests that SCP-6862 is forged from the curiosity of a grand deity, wrought with tools beyond the common understanding of humanity, oddities to be harnessed by those most powerful.

In order to prevent further 'pataphysical involvement, we simply made it stop any outside entities from rewriting narratives and deleting these files. We hope that this will cease any further misdemeanors by REDEYE, allowing it to function as a successful containment procedure for 8420. So the only file it has too much access in is SCP-8420 itself.

- Senior Researcher Octavia Cleary,
RT-8420
