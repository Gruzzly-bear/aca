To view the digital documents within this exclusionary server partition, you are required to test your device using the in-built detection system no matter your station or clearance.

The window below this message is designed to both detect the type of device you are using and its current state:

  • If you are using a Foundation-issued PC, the window below should either show "PC Wide" or "PC Thin". Adjust the width of the application window you are using to confirm that both states are being detected. Then return the application window to the "PC Wide" state.
    • The commonly in-built "Inspect Element" browser function can be an easier method of doing this.
  • If you are using a Foundation-issued mobile device, the window below should either show "Mobile Portrait" or "Mobile Landscape". Rotate your device to confirm that both states are being detected. Then return your device to the "Mobile Portrait" state

— Maria Jones, Director, RAISA

An Exclusionary Partition? Why?

I thought those were only used when there's concerns for something corrupting the documents such as timeline fuck-ups, other anomalies, or… something getting out of them…

This does not bode well.

We have not been detected by RAISA yet; we can log off and forget this ridiculous plan of yours.

Don't care Ros, we're doing this.

We're already barking up a "Hippomane mancinella", but we need to know what the Foundation is hiding from us… we need to know if we're a danger to everyone around us.

…Satisfactory; Although

It is curious as to why there is a system in place to detect those specific device variables. What could window width or device orientation have to do with reading documents?

Yea, it is strange…

Usually there's only a system in place to differentiate between civilian and Foundation-issued devices… perhaps they're trying to prevent a work-around in the system from being exploited?
